# ADA Final Project Delivery Package

This folder separates final/submission-facing materials from working outputs in `project/output/`.

## Folder Map

| Folder | Purpose | Main Files |
| --- | --- | --- |
| `01_report/` | Report draft and report-format source files | report PDF snapshot, report Markdown, LaTeX source, bibliography |
| `02_valuation_model/` | Valuation workbooks and key model data | Excel model, beautified workbook, key processed CSVs |
| `03_presentation_support/` | Presentation preparation and personal review materials | presentation guide, glossary, human review checklists |
| `04_ai_supplement/` | AI-use documentation and code required for reproducibility | AI usage log, data inventory, Python/JS scripts |
| `05_dashboard/` | Static web dashboard files for GitHub Pages or local viewing | `index.html`, CSS, JS, dashboard data |

## Recommended Submission Set

Use this set if a compact final submission is needed:

1. Report: `01_report/lululemon_valuation_report_snapshot.pdf`
2. Model: `02_valuation_model/lulu_valuation_model_beautified.xlsx`
3. AI usage record: `04_ai_supplement/AI_USAGE_LOG.md`
4. Code: `04_ai_supplement/code/`
5. Key data: `02_valuation_model/key_data/`
6. Dashboard, if used as a supplementary exhibit: `05_dashboard/`

## Notes

- The PDF report snapshot is an archive/export of the current report draft. The human-written final report should still be reviewed before submission.
- The presentation slides and final speech script are not generated here because they are expected to be written by the presenting students.
- `project/output/` remains the working output folder for generated screenshots, HTML exports, previews, and intermediate files.
- Raw SEC files are kept in `project/data/raw/` and are not duplicated here to keep the delivery package manageable.
